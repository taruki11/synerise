# Synerise 项目客观定位与投递建议（2026-04-13）

## 结论先说

这个项目不是“很水”，也不是“拿出去乱杀”的顶级算法项目。它的客观定位更接近：

- 一份扎实的、偏数据挖掘与经营建模的时序用户建模项目
- 强项在于大规模日志处理、用户级特征工程、时间切分、Top-K 与策略分析
- 短板在于没有真实线上环境、没有深度模型创新、没有论文级方法亮点

如果你讲得稳，它足以支撑：

- 数据挖掘算法实习
- 用户增长 / CRM / 留存建模实习
- 机器学习算法实习（偏 tabular / 业务建模）
- 电商策略算法 / 经营分析算法实习

如果你想靠它硬冲：

- 推荐排序核心算法
- 搜索排序核心算法
- 以大模型 / 深度学习 / 顶会论文为主的算法岗

那就不够强。

## 为什么我说它不是很水

### 1. 数据和工程明显强于普通 churn 项目

官方 Synerise 挑战说明明确写了挑战围绕统一用户行为建模展开，日志包括购买、加购、移除购物车、页面访问和搜索行为，数据以 Parquet 形式提供，并包含 churn open task。[官方 Summary](https://recsys.synerise.com/summary) [官方 GitHub](https://github.com/Synerise/recsys2025)

这说明你的项目不是拿一个几千行 CSV 做课堂练习，而是基于官方大规模行为日志做的下游建模。

### 2. 公开同类 churn 项目大多更简单

GitHub 上 `customer-churn-prediction` 主题下有大量公开项目，但大多数是小数据集、Jupyter Notebook、随机切分、传统分类器和简单 EDA。[GitHub Topics - customer-churn-prediction](https://github.com/topics/customer-churn-prediction)

跟这些项目相比，你现在这份项目的优势是：

- 原始日志规模更大
- 用了 DuckDB 处理 parquet
- 有严格时间切分
- 有 rolling backtest
- 有 Top-K 策略分析
- 现在还补了 ROI 模拟和 A/B 设计

所以它显著高于“普通 Kaggle churn notebook”。

## 为什么它也不是特别顶

### 1. 不是官方比赛标准交付物

Synerise 官方 challenge 要提交的是用户表示 embedding，而不是直接提交 churn 概率。官方页面和仓库都明确写了，参赛者提交的是 universal behavioral profiles，组织方再拿这些表示去训练下游任务模型。[官方 Summary](https://recsys.synerise.com/summary) [官方 GitHub](https://github.com/Synerise/recsys2025)

所以你现在这份项目更像：

- “基于官方数据集独立完成的 churn prediction 项目”

而不是：

- “官方挑战标准参赛结果”

### 2. 算法深度偏业务建模，不偏前沿方法

你当前主力方法还是：

- 用户级聚合特征
- XGBoost / LightGBM
- 时间切分评估

这对实习项目是合理的，但不属于方法创新很强的项目。

### 3. 没有真实线上闭环

即使现在补了 ROI 模拟和 A/B 设计，它们仍然是：

- 离线模拟
- 上线方案

不是已经落地的真实线上实验。

## 这个项目最适合投什么方向

### 第一梯队：最匹配

- 数据挖掘算法实习
- 用户增长 / 用户建模 / 留存策略实习
- 电商经营分析算法实习
- CRM / 流失预测 / 用户价值建模实习

原因：这个项目的主线和这些岗位的工作内容高度一致。

### 第二梯队：可以投

- 通用机器学习算法实习
- 风控 / 策略算法实习
- 工业 AI 里偏时序与表格建模的岗位

原因：你能展示完整的机器学习建模流程和时序评估意识。

### 第三梯队：可以投，但不要主打它

- 推荐算法实习
- 搜索排序实习

原因：当前项目不是排序问题，没有召回、排序、在线多目标优化、曝光反馈、样本偏差校正等核心内容。

### 第四梯队：不建议靠它硬冲

- 以大模型为主的算法实习
- 以论文和深度学习为主的科研算法岗

原因：这项目的亮点不在深度模型和研究创新。

下面这个分数不是绝对水平，而是“作为单个面试项目”的参考。

- 通用机器学习实习：`7.5/10`
- 电商策略 / 经营分析算法：`8/10`
- 推荐算法：`6/10`

## 你这个项目最容易被认可的点

- 数据规模大，不是玩具数据
- 时间切分意识强
- 特征工程主线清楚
- 用 DuckDB 解决大表聚合，有工程味
- strict 版比旧版更合理
- 现在补了 Top-K、ROI 模拟和 A/B 设计，业务闭环更完整

## 最容易被质疑的点

- 不是官方比赛标准提交
- 没有真实线上实验
- 方法上不够“炫”
- 如果你自己讲不清楚，会显得像“别人帮你堆出来的项目”

## 我对你现在的真实建议

### 简历上怎么用

可以把它放第一项目，尤其是投数据挖掘、用户增长、经营建模方向时。

### 面试里怎么用

如果你现在对代码和逻辑已经能讲住，那它可以做主讲项目。  
如果你还觉得自己不够稳，就把它当第一项目写简历、第二项目讲面试。

### 最吃香的投递方向

按你当前这项目和你已有背景，最值得优先投的是：

1. 数据挖掘算法实习
2. 用户增长 / 用户建模 / 留存策略实习
3. 电商经营分析 / CRM 建模实习
4. 通用机器学习算法实习
5. 工业 AI / 时序建模实习

推荐算法和大模型岗不是不能投，但不要把希望主要压在这个项目上。

## 一句话最终结论

这个项目是“对业务建模岗很能打、对推荐排序岗一般、对研究型算法岗偏弱”的项目。只要你讲得稳，它完全够你投一批高质量的数据挖掘、用户增长和机器学习实习岗位。
